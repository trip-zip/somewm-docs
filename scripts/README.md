# Reference generators

Scripts that produce the long enumerated reference pages from upstream sources of truth.

## When to run

- Before cutting a docs release.
- After a SomeWM release that adds, renames, or removes `@beautiful` LDoc tags.
- After bumping the system `libxkbcommon` version (rare; the keysym list grows slowly).

## How to run

```bash
npm run generate:reference
```

This invokes `scripts/regenerate-all.mjs`, which:

1. Creates temporary `git worktree` checkouts of `/home/jimmy/tools/some` at `main` and `release/1.4`.
2. Runs `generate-theme-variables.mjs` against each worktree.
3. Runs `generate-keysyms.mjs` once against `/usr/include/xkbcommon/xkbcommon-keysyms.h` and copies the output into both doc versions.
4. Removes the temporary worktrees.

After it finishes, review the diff (`git diff --stat docs/ versioned_docs/`) and commit if the changes look correct.

## Outputs

| Page | Generated by | Inputs |
| --- | --- | --- |
| `docs/reference/beautiful/theme-variables.md` | `generate-theme-variables.mjs` | `@beautiful` LDoc tags + Lua files referencing `beautiful.*` |
| `versioned_docs/version-1.4/reference/beautiful/theme-variables.md` | same | same, from `release/1.4` branch |
| `docs/reference/key-names-all.md` | `generate-keysyms.mjs` | `/usr/include/xkbcommon/xkbcommon-keysyms.h` |
| `versioned_docs/version-1.4/reference/key-names-all.md` | (copy) | identical to above |

## Hand-curated companions

These are **not** generated and survive regeneration:

- `docs/reference/beautiful/_theme-variables-prelude.md` – inlined into the top of the theme variables page before the auto-generated module sections. Edit this to add introductory prose, quick-start tables, or callouts.
- `docs/reference/key-names.md` – the curated common-keys reference. Edit normally.

The 1.4 versioned copies of the above must be edited separately.

## Running a single generator

```bash
node scripts/generate-theme-variables.mjs \
  --source /home/jimmy/tools/some \
  --prelude docs/reference/beautiful/_theme-variables-prelude.md \
  --out docs/reference/beautiful/theme-variables.md \
  --ref main

node scripts/generate-keysyms.mjs \
  --header /usr/include/xkbcommon/xkbcommon-keysyms.h \
  --out docs/reference/key-names-all.md
```

## Updating the sync log

After regenerating, append a line to `ideas/sync-log.md` with the SomeWM SHA the docs are now in sync with. The `Last sync:` line in each generated page records this automatically; the log captures it at the repo level.
